## 🏆 What Makes Us Unique — The Proof of Work Layer

Every competing team will use GPS + weather data for fraud detection. We go further.

### The Problem With GPS-Only Fraud Detection

```
GPS says: "Worker is in the disruption zone"  ✓
But doesn't answer: "Did the worker ACTUALLY try to work?"

A fraudster can:
  • Leave their phone in a flood zone while staying home
  • Use a GPS spoofing app to fake location
  • File claims from bed with a stationary device
```

### Our Solution: Proof of Work (PoW)

Inspired by how Bitcoin verifies miners *actually did computational work*, our PoW layer verifies workers *actually tried to deliver*.

```
┌────────────────────────────────────────────────────────────┐
│                  PROOF OF WORK ENGINE                      │
│                                                            │
│  Signal 1: Order Acceptance Logs             (30% weight) │
│  ──────────────────────────────────────────────────────   │
│  Did the worker attempt to accept orders?                 │
│  Genuine:   3–8 attempts → platform auto-cancelled        │
│  Fraudster: 0 attempts (no reason to try if faking)       │
│                                                            │
│  Signal 2: Movement Pattern Analysis         (25% weight) │
│  ──────────────────────────────────────────────────────   │
│  Did the worker move toward a pickup location?            │
│  Genuine:   stop-start navigation, 0.5–2km movement      │
│  Fraudster: stationary OR scripted teleportation          │
│                                                            │
│  Signal 3: App Interaction Patterns          (25% weight) │
│  ──────────────────────────────────────────────────────   │
│  Did the worker interact with the app naturally?          │
│  Genuine:   10–20 taps, variable timing, human variance  │
│  Bot/Fraud: perfect intervals, no natural hesitation      │
│                                                            │
│  Signal 4: Nearby Worker Density Correlation (20% weight) │
│  ──────────────────────────────────────────────────────   │
│  Are nearby workers also idle during this event?          │
│  Genuine:   60–100 nearby workers show same pattern      │
│  Isolated:  worker anomalously idle vs all peers          │
│                                                            │
│  PoW = (Orders×0.30)+(Movement×0.25)                     │
│       +(App Activity×0.25)+(Density×0.20)                │
│                                                            │
│  Score > 70 → Genuine worker, fast-track approval        │
│  Score < 30 → Suspected fraud, hold + manual review      │
└────────────────────────────────────────────────────────────┘
```

**Why this wins:** No insurance platform in India — startup or enterprise — currently uses behavioral micro-signals to verify genuine work attempts. This is novel, defensible, and directly solves the #1 risk in gig insurance: coordinated fraud rings.
